​Change E-commerce Status Request

POST
https://uat-api.enza.cloud/TX/v1
/cards/tdsenrollement

This request is used to modify the e-commerce status associated with card transactions by providing ECstatus and the card reference number.

​Headers

  • x-api-key
    string
    required
  • Content-Type
    string
    required

​Request Body

  • CardRefNum
    string

    Card Reference Number.

  • ECstatusEnable
    string

    ECstatus ( "1" -> Enrolled, "0" -> Not enrolled).

​Responses

    • message
      string

      Success! ECstatus is successfully changed!!(Enabled)!

POST /cards/tdsenrollement

​Change Card Status

POST
https://uat-api.enza.cloud/TX/v1
/cards/cardstatus

Update the status of a card by providing the new status and the card reference number. This endpoint allows the modification of the card's current status to be "Active" or "Deactivated".

​Headers

  • x-api-key
    string
    required
  • Content-Type
    string
    required

​Request Body

  • CardStatus
    string

    The status of the card.

  • CardRefNum
    string

    Card Reference Number.

​Responses

    string
POST /cards/cardstatus
